## 1. 系统概述

本系统是一款面向A股市场的日线级别交易信号分析工具，核心功能包括：

- **多策略信号生成**：内置4种独立交易策略，各策略独立产生买卖信号
- **策略融合**：通过加权投票机制将多策略信号融合为最终信号
- **历史回测**：模拟A股T+1交易规则，含佣金/印花税/涨跌停处理，支持交易记录查看和未平仓处理
- **参数优化**：支持网格搜索和遗传算法两种优化方式，支持优化结果保存、加载和应用
- **K线图可视化**：基于mplfinance的K线图，支持信号标注、交易记录标注、技术指标叠加
- **配置管理**：支持策略配置保存和加载，可保存全局和个股专用配置

## 4. 数据管理

### 4.1 数据源

系统采用多数据源自动切换机制，确保数据获取的稳定性：

| 数据类型 | 优先级 | 数据源 | 接口 |
|---------|--------|--------|------|
| 历史日线 | 1 | 腾讯 | `stock_zh_a_hist_tx` |
| 历史日线 | 2 | 新浪 | `stock_zh_a_daily` |
| 历史日线 | 3 | 东方财富 | `stock_zh_a_hist` |
| 股票列表 | 1 | 交易所官方 | `stock_info_a_code_name` + 深交所/上交所 |
| 股票列表 | 2 | 新浪 | `stock_zh_a_spot` |
| 股票列表 | 3 | 东方财富 | `stock_zh_a_spot_em` |

当高优先级数据源不可用时，系统自动切换到下一个数据源。

### 4.2 数据格式

系统内部统一数据格式：

| 字段 | 类型 | 说明 |
|------|------|------|
| symbol | str | 股票代码（纯数字，如 "000001"） |
| date | str | 日期（格式 "YYYY-MM-DD"） |
| open | float | 开盘价 |
| high | float | 最高价 |
| low | float | 最低价 |
| close | float | 收盘价 |
| volume | float | 成交量（股） |
| amount | float | 成交额（元） |
| turnover | float | 换手率（%） |

### 4.3 数据存储

- 数据库：SQLite，存储路径 `data/stock_data.db`
- 数据表：`daily_kline`（日线数据）、`stock_info`（股票信息）
- 缓存：股票列表在内存中缓存，避免重复请求

### 4.4 增量更新

系统支持增量更新，仅获取数据库中最新日期之后的新数据，减少网络请求量。

## 5. 策略体系

系统内置4种独立交易策略，每种策略从不同维度分析市场：

### 5.1 技术指标策略 (TechIndicatorStrategy)

**核心逻辑**：综合MACD、KDJ、RSI、布林带等经典技术指标产生信号。

**买入条件**（满足以下多数条件）：
- MACD金叉（DIF上穿DEA）
- KDJ金叉（K线上穿D线）且K < 80
- RSI(6) < 30（超卖区域）
- 收盘价触及布林带下轨

**卖出条件**（满足以下多数条件）：
- MACD死叉（DIF下穿DEA）
- KDJ死叉（K线下穿D线）且K > 20
- RSI(6) > 70（超买区域）
- 收盘价触及布林带上轨

**可调参数**：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| macd_fast | 12 | — | MACD快线周期 |
| macd_slow | 26 | — | MACD慢线周期 |
| macd_signal | 9 | — | MACD信号线周期 |
| kdj_n | 9 | — | KDJ的N日周期 |
| kdj_m1 | 3 | — | K值平滑系数 |
| kdj_m2 | 3 | — | D值平滑系数 |
| rsi_periods | [6,12,24] | — | RSI计算周期列表 |
| rsi_buy_threshold | 30 | — | RSI买入阈值 |
| rsi_sell_threshold | 70 | — | RSI卖出阈值 |
| boll_period | 20 | — | 布林带周期 |
| boll_std | 2 | — | 布林带标准差倍数 |

### 5.2 均值回归策略 (MeanReversionStrategy)

**核心逻辑**：价格偏离均线后回归均值，结合支撑阻力位判断。

**买入条件**：
- 价格偏离均线下方超过阈值
- 偏离度开始缩小（前一日偏离更大）
- 接近支撑位时信号更强

**卖出条件**：
- 价格偏离均线上方超过阈值
- 偏离度开始缩小
- 接近阻力位时信号更强

**可调参数**：

| 参数 | 默认值 | 范围 | 步长 | 说明 |
|------|--------|------|------|------|
| ma_period | 20 | 10-30 | 5 | 均线周期 |
| deviation_threshold | 2.0 | 1.5-3.0 | 0.5 | 偏离阈值(%) |
| support_lookback | 20 | 10-30 | 5 | 支撑位回看天数 |
| resistance_lookback | 20 | 10-30 | 5 | 阻力位回看天数 |
| reversion_speed | 0.5 | 0.3-0.8 | 0.1 | 回归速度系数 |

### 5.3 量价分析策略 (VolumePriceStrategy)

**核心逻辑**：成交量异动配合价格变化和OBV趋势确认。

**买入条件**：
- 量比 > 阈值（放量）
- 价格上涨超过阈值
- OBV呈上升趋势

**卖出条件**：
- 量比 > 阈值（放量）
- 价格下跌超过阈值
- OBV呈下降趋势

**可调参数**：

| 参数 | 默认值 | 范围 | 步长 | 说明 |
|------|--------|------|------|------|
| volume_ratio_threshold | 2.0 | 1.5-3.0 | 0.5 | 量比阈值 |
| volume_ma_period | 5 | 3-10 | 1 | 量比均线周期 |
| price_change_threshold | 0.03 | 0.02-0.05 | 0.01 | 价格变化阈值 |
| obv_trend_period | 10 | 5-15 | 5 | OBV趋势判断周期 |

### 5.4 趋势跟踪策略 (TrendFollowingStrategy)

**核心逻辑**：均线金叉/死叉配合动量确认和ATR止损参考。

**买入条件**：
- 快均线上穿慢均线（金叉）
- 价格在趋势均线之上
- 动量（N日收益率）> 0

**卖出条件**：
- 快均线下穿慢均线（死叉）
- 价格在趋势均线之下
- 动量 < 0

**可调参数**：

| 参数 | 默认值 | 范围 | 步长 | 说明 |
|------|--------|------|------|------|
| fast_ma | 5 | 3-10 | 1 | 快均线周期 |
| slow_ma | 20 | 15-30 | 5 | 慢均线周期 |
| trend_ma | 60 | 40-80 | 10 | 趋势均线周期 |
| momentum_period | 10 | 5-15 | 5 | 动量计算周期 |
| atr_period | 14 | 10-20 | 5 | ATR周期 |

## 6. 策略融合

### 6.1 融合机制

系统采用**加权投票**方式融合多策略信号：

1. 各策略独立产生买卖信号
2. 按日期汇总所有信号
3. 计算买入得分和卖出得分：

```
买入得分 = Σ(策略权重 × 信号置信度 × 信号强度)
卖出得分 = Σ(策略权重 × 信号置信度 × 信号强度)
净得分 = 买入得分 - 卖出得分
```

4. 根据净得分判定最终信号：
   - 净得分 ≥ 买入阈值(0.6) → **买入**
   - 净得分 ≤ 卖出阈值(-0.6) → **卖出**
   - 其他 → **持有**

5. 信号强度由达成一致的策略数量决定：
   - 3个及以上策略一致 → **强信号**
   - 2个策略一致 → **中等信号**
   - 1个策略 → **弱信号**

### 6.2 默认权重

| 策略 | 默认权重 |
|------|---------|
| 技术指标 | 0.25 |
| 均值回归 | 0.25 |
| 量价分析 | 0.25 |
| 趋势跟踪 | 0.25 |

### 6.3 权重调整

可在"策略配置"Tab页中调整各策略权重，也可通过回测结果自动调整（按各策略胜率分配权重）。

### 6.4 配置保存与加载

**保存配置**：
- 调整好策略参数和融合权重后，点击"保存配置"按钮
- 配置将保存到数据库
- 如果已选中股票，保存为该股票的专用配置
- 如果未选中股票，保存为全局配置

**加载配置**：
- 点击"加载配置"按钮
- 系统从数据库读取并应用配置
- 界面自动更新，配置立即生效

**应用配置**：
- 点击"应用配置"按钮
- 将当前界面的配置应用到策略引擎
- 不会保存到数据库，仅本次会话有效

**重置默认**：
- 点击"重置默认"按钮
- 恢复为 config.yaml 中定义的默认参数

## 7. 回测系统

### 7.1 回测流程

```
获取数据 → 策略生成信号 → 信号融合 → 模拟交易 → 绩效分析
```

### 7.2 交易模拟规则

系统严格模拟A股交易规则：

| 规则 | 说明 |
|------|------|
| T+1 | 当日买入的股票次日才能卖出 |
| 涨跌停 | 涨停日无法买入，跌停日无法卖出 |
| 交易单位 | 最小100股（1手），按手整数倍交易 |
| 佣金 | 万三（0.03%），最低5元 |
| 印花税 | 千一（0.1%），仅卖出时收取 |
| 全仓模式 | 买入时使用全部可用资金 |

### 7.3 回测参数

在"回测"Tab页中设置：

- **股票代码**：直接输入或使用当前选中股票
- **策略选择**：技术指标 / 均值回归 / 量价分析 / 趋势跟踪 / 融合策略
- **起止日期**：回测时间范围
- **初始资金**：默认100,000元

### 7.4 回测结果指标

| 指标 | 说明 |
|------|------|
| 总交易次数 | 回测期间完成的买卖交易对数 |
| 盈利/亏损次数 | 盈利和亏损的交易次数 |
| 胜率 | 盈利交易次数 / 总交易次数 |
| 平均盈亏率 | 所有交易的平均收益率 |
| 盈亏比 | 平均盈利幅度 / 平均亏损幅度 |
| 最大回撤 | 权益曲线从峰值到谷值的最大跌幅 |
| 最大回撤持续 | 最大回撤持续的交易日数 |
| 总收益率 | (期末权益 - 期初权益) / 期初权益 |
| 年化收益率 | 总收益率按252个交易日年化 |
| 夏普比率 | (年化收益 - 无风险利率) / 收益率标准差 × √252 |

### 7.5 回测报告示例

```

策略: tech_indicator
时间: 2023-01-01 ~ 2024-12-31
参数: {macd_fast: 12, macd_slow: 26, ...}

总交易次数: 15
盈利次数: 9
亏损次数: 6
胜率: 60.00%
平均盈亏率: 2.35%
盈亏比: 1.85
最大回撤: 8.50%
最大回撤持续: 12天
总收益率: 25.30%
年化收益率: 12.65%
夏普比率: 1.35
```

### 7.6 交易记录查看

回测完成后，系统会显示详细的交易记录表格，包含：

| 字段 | 说明 |
|------|------|
| 入场日期 | 买入日期 |
| 入场价格 | 买入价格 |
| 出场日期 | 卖出日期（未平仓时显示为 "YYYY-MM-DD(未平仓)"） |
| 出场价格 | 卖出价格 |
| 盈亏 | 交易盈亏金额（元） |
| 盈亏率 | 交易收益率（%） |
| 持仓天数 | 持仓交易日数 |

**未平仓处理**：
- 回测结束时，如果还有未卖出的持仓，系统会自动：
- 使用回测最后一日收盘价计算未实现盈亏
- 不计入回测最后一日作为出场日期（标记未平仓）
- 在权益计算到绩效指标计算中包含未平仓

### 7.7 交易标注到K线图

点击回测面板中的"标注交易到K线图"按钮，可将回测的交易记录标注在K线图上：

- 红色圆点🔴：入场（买入）
- 绿色方块🟢：出场（卖出）

同时会自动切换到K线图标签页。

## 8. 参数优化

### 8.1 网格搜索 (Grid Search)

**原理**：遍历参数空间中所有可能的组合，逐一回测，选择最优参数。

**配置项**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| max_combinations | 500 | 最大搜索组合数 |

**适用场景**：参数空间较小、需要全面搜索时使用。

### 8.2 遗传算法 (Genetic Algorithm)

**原理**：模拟自然选择过程，通过选择、交叉、变异操作迭代优化参数。

**配置项**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| population_size | 50 | 种群大小 |
| generations | 30 | 迭代代数 |
| crossover_rate | 0.7 | 交叉概率 |
| mutation_rate | 0.2 | 变异概率 |
| tournament_size | 3 | 锦标赛选择大小 |

**适用场景**：参数空间较大、需要高效搜索时使用。

### 8.3 优化目标

可选择以下指标作为优化目标：

| 指标 | 说明 |
|------|------|
| 夏普比率 | 风险调整后收益（推荐） |
| 胜率 | 盈利交易占比 |
| 总收益率 | 绝对收益水平 |
| 盈亏比 | 平均盈利与平均亏损之比 |

### 8.4 使用步骤

1. 在"参数优化"Tab页选择策略和优化方法
2. 选择优化目标指标
3. 设置优化参数（种群大小、代数等）
4. 点击"开始优化"
5. 优化完成后查看最优参数
6. 可将最优参数应用到策略配置中

### 8.5 优化结果管理

**保存优化结果**：
- 优化完成后，结果自动保存到数据库
- 保存内容包括：股票代码、股票名称、优化方法、优化目标、最优参数、优化时间等

**加载历史优化结果**：
- 点击"加载历史"按钮
- 查看该股票的所有历史优化结果
- 可选择任意一条查看详情

**应用优化参数**：
- 选中某条优化结果
- 点击"应用参数"按钮
- 最优参数自动填充到策略配置面板
- 自动切换到策略配置标签页

## 9. 信号生成与查看

### 9.1 生成信号

1. 在左侧面板选择一只股票
2. 切换到"信号"Tab页
3. 点击"生成信号"按钮
4. 系统将使用所有启用的策略生成融合信号

### 9.2 信号属性

每个信号包含以下信息：

| 属性 | 说明 |
|------|------|
| 日期 | 信号产生日期 |
| 类型 | 买入(BUY) / 卖出(SELL) / 持有(HOLD) |
| 强度 | 强(STRONG) / 中(MEDIUM) / 弱(WEAK) |
| 价格 | 信号日收盘价 |
| 置信度 | 0-1之间的信号可信度 |
| 贡献策略 | 产生该信号的策略列表 |
| 描述 | 信号的文字描述 |

### 9.3 信号筛选与排序

**信号筛选**：
- 按股票代码筛选：输入股票代码，仅显示该股票的信号
- 按信号类型筛选：选择"全部"、"买入"、"卖出"、"持有"

**信号排序**：
- 点击任意列标题进行排序
- 再次点击同一列切换升序/降序
- 支持排序的列：证券代码、日期、信号类型、强度、置信度、价格

### 9.4 历史信号加载

点击"加载历史信号"按钮，可从数据库读取之前保存的信号。

### 9.5 标注到K线图

在信号面板中点击"显示在K线图"按钮，信号将以箭头标注在K线图上：
- 买入信号：红色向上箭头▲
- 卖出信号：绿色向下箭头▼

同时可配合回测的交易记录标注一起查看（红色圆点为入场，绿色方块为出场）。

## 12. 常见问题

### Q1: 启动后搜索股票失败？

**原因**：网络连接问题或数据源接口暂时不可用。

**解决方案**：系统会自动切换数据源（腾讯→新浪→东方财富），如全部失败则使用内置默认股票列表。请检查网络连接后重试。

### Q2: K线图不显示？

**原因**：该股票数据尚未下载。

**解决方案**：在左侧面板选中股票后，系统会自动从数据源获取数据。如获取失败，请检查网络连接。

### Q3: 回测结果为空？

**原因**：在选定的时间范围内，策略未产生任何买卖信号。

**解决方案**：
- 扩大回测时间范围
- 调整策略参数（如降低买卖阈值）
- 尝试其他策略

### Q4: 参数优化耗时过长？

**解决方案**：
- 减少网格搜索的最大组合数
- 使用遗传算法代替网格搜索
- 缩小参数范围
- 缩短回测时间范围

### Q5: 如何切换深色/浅色主题？

修改 `config.yaml` 中的 `ui.theme` 字段：
- `dark`：深色主题
- `light`：浅色主题

修改后重启程序生效。

### Q6: 数据存储在哪里？

- 行情数据：`data/stock_data.db`（SQLite数据库）
- 日志文件：`logs/` 目录
- 配置文件：`config.yaml`

### Q7: 如何重置所有数据？

删除以下文件后重启程序：
- `data/stock_data.db`
- `logs/` 目录下所有文件

程序会自动重新创建数据库和日志。

### Q8: 为什么信号面板显示的信号数量和回测的交易次数不一样？

这是正常的设计！**信号生成**和**回测处理**存在以下差异：

| 项目 | 信号生成 | 回测处理 |
|------|---------|---------|
| HOLD信号 | 显示 | 跳过 |
| 涨跌停 | 不检查 | 检查（涨停买不到，跌停卖不出） |
| T+1规则 | 不考虑 | 当日买入不能当日卖出 |
| 持仓状态 | 不检查 | 空仓才买入，持仓才卖出 |

因此，回测交易次数通常会比信号面板显示的信号数量少，这更符合真实市场情况。

### Q9: 回测结果中有"未平仓"标记是什么意思？

表示回测结束时，策略仍然持有该股票尚未卖出。系统会：
- 使用回测最后一天的收盘价计算未实现盈亏
- 将此部分收益包含在总收益率等绩效指标中
- 出场日期标记为 "YYYY-MM-DD(未平仓)"

### Q10: 如何为不同股票保存不同的策略配置？

1. 先在左侧面板选中一只股票
2. 在策略配置面板调整好参数
3. 点击"保存配置"
4. 配置将保存为该股票专用配置
5. 下次选中该股票时，点击"加载配置"即可恢复

未选中股票时保存的是全局配置，所有股票通用。
